# 入力要約コマンド

`docs/00_inputs/` 配下の全 `.md` ファイルを要約し、`docs/00_inputs/summary.md` を生成。

## 実行手順

1. `docs/00_inputs/**/*.md` を読み込み（`summary.md` 自体は除外）
2. 以下の構造で要約を作成:
   - プロジェクト概要
   - ビジネス背景/課題
   - 要件・制約
   - 利用者・ステークホルダー
   - 業務フロー/シナリオ
   - その他の参考情報
3. **Write ツール**で `docs/00_inputs/summary.md` に保存

## 要約の粒度

- 目標: 2,000〜4,000トークン程度
- ドラフターがこの要約のみで仕様書等を書けるレベル
- 重複は統合、詳細な議事録は結論のみ抽出
