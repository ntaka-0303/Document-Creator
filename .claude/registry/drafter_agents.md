# ドラフタースタイルレジストリ

## 利用可能なスタイル

| スタイルID | 特徴 | 主な用途 |
|-----------|------|----------|
| structured | 章立て厳守、網羅性重視 | 全ドキュメント（基盤） |
| narrative | ストーリー性、読み物として | 提案書、ワークフロー |
| impact | 結論ファースト、経営層向け | 提案書、サマリ |
| concise | 最小限、箇条書き中心 | 仕様書、計画書 |
| logical | WHY/WHAT/HOW、因果明確 | 仕様書、要件定義書 |

## キーワードによる自動選択

| ユーザー発言 | 選択スタイル |
|-------------|-------------|
| ストーリー、読み物 | narrative |
| 経営層、インパクト | impact |
| 簡潔、短く | concise |
| 論理構成、Why/What/How | logical |

## 統合ドラフターエージェント

すべてのスタイルは `doc-drafter` エージェントで統一処理。
`style` パラメータで出力形式を切り替え。
