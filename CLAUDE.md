# DocumentCreator プロジェクトガイド

## 言語設定（必須）

**すべての応答は必ず日本語で行うこと。**

## コマンド実行時の振る舞い

### `/create_document` スキル実行時

スキル `/create_document` として実行されます。

1. **説明や挨拶は不要** - 即座にワークフローを開始
2. **最初のアクションはドキュメント種別の選択**:

```
作成するドキュメント種別を選択してください：

1. 提案書
2. 計画書
3. 要件定義書
4. To-Beワークフロー / 業務フロー
5. 機能仕様書 / システム仕様書

番号を入力してください（1-5）:
```

## プロジェクト概要

AIによるドキュメント自動生成システム。

**エージェント構成:**
- `doc-drafter`: 統合ドラフター（style パラメータで出力スタイルを切替）
- `doc-evaluator`: 評価エージェント
- `doc-finalizer`: 最終化エージェント

**モード体系（3モード）:**
1. **標準モード（デフォルト）**: 1ドラフター → 自己レビュー → 完了
2. **品質モード**: 1ドラフター → 評価 → 改善 → 完了
3. **高品質モード**: 2ドラフター → 評価 → 統合 → 完了

## ディレクトリ構造

```
docs/
├── 00_inputs/           # 入力情報（summary.md を生成）
├── 01_proposal/         # 提案書
├── 02_plan/             # 計画書
├── 03_requirement/      # 要件定義書
├── 04_to-be_workflow/   # To-Beワークフロー
└── 05_spec/             # 仕様書
    ├── definitions/
    │   ├── template.md  # テンプレート
    │   └── evaluation.md # 評価基準
    └── output/          # 最終版出力先
```

## ファイル保存ルール

- **Write ツールを使用**（`@save` 記法は使用しない）
- **最終版パス**: `docs/<doc_type>/output/<filename>_<YYYYMMDD>.md`
- **中間ドラフトはメモリ上で管理**（ファイル保存しない）
