# **3. 本提案の概要（エグゼクティブサマリー）**

* 本提案は、東亜セキュリティーズ株式会社様における  
  　**リテール・法人営業を中心とした「文書起点の知的業務」をAIで自動化・高度化し、営業生産性とナレッジ活用力を飛躍的に向上すること** を目的としています。
* LayerX が提供する **AIWorkforce（エンタープライズ向け AIWorkforce プラットフォーム）** を活用し、  
  　市場情報・投信レポート・社内ファイル・過去提案書などに分散したナレッジを統合し、  
  　**「検索 → 分析 → 提案資料作成 → レビュー」の一連の業務フローをAIエージェントで支援・自動化** します。
* 本提案のポイント
  - リテール営業企画部・法人企画部・DX推進部・情シスを横断した **共通AI基盤** の整備
  - **ナレッジ検索の統合**（市場情報／投信レポート／社内ファイルの一括検索）
  - **提案資料の自動作成**（顧客属性 × 市況情報 × 商品解説を自動生成）
  - **法人営業の提案書・案件情報のデータレイク化** による再利用性向上
  - 支店スタッフ向け **高度FAQボット** による問い合わせ対応の効率化
  - Azure 上の顧客専用環境（Single-Tenant / 顧客テナント）による金融機関レベルのセキュリティ

# **4. 課題認識**

## 4.1. 現状の業務課題（東亜セキュリティーズ様の状況）

1. **個人リテール営業の生産性の低さ**
   * 店舗営業スタッフが、提案資料作成・市場情報整理に多くの時間を費やしている
   * 金融情報・市場ニュース・投信レポートが複数システムに分散し、**検索性が低い**
   * 顧客ヒアリング内容から提案ストーリーを組み立てるプロセスが属人化しており、営業担当者ごとの品質差が大きい

2. **法人営業におけるナレッジ蓄積・再利用の不足**
   * 過去提案書や案件情報がファイルサーバ／個人PC／部門ごとのツールに散在
   * 類似案件のベストプラクティスが検索しづらく、**毎回スクラッチで提案書を作る非効率** が発生

3. **営業支援・情報系ツールの乱立**
   * CRM、提案管理、商品情報閲覧、調査部レポート閲覧など、  
     **10以上のシステムを横断的に参照する必要があり、現場負荷が高い**
   * 部門ごとに部分最適なツール導入が進んだ結果、全体としてのUX・データ連携が分断されている

4. **情報システム部（情シス）の負荷増大とガバナンス課題**
   * 各部門から「AIを使って業務を効率化したい」という相談・PoC要望が殺到
   * 部門単位での個別PoCが乱立し、**セキュリティ・コスト・運用面でスケールしない** 状況
   * レガシーシステムと最新クラウドが混在し、**データ連携・統制ポリシーに制約が多い**

## 4.2. 経営環境の変化

* 金融業界全体での **AI・クラウド活用競争の加速** により、  
  営業生産性・提案力・ナレッジ活用力の差が競争力の差に直結
* 金融庁等による **ガバナンス・リスク管理要件の高度化** に対応しながら、  
  コスト削減とDX推進を両立することが求められている
* 顧客接点のデジタル化（非対面チャネルの拡大）により、  
  **高品質な提案資料や説明コンテンツを迅速に提供できるかどうか** が重要な差別化要因となっている

# **5. 解決コンセプト**

本提案では、AIWorkforce を中核とした **「AIエージェント × ワークフロー × ナレッジポータル」** による  
**営業・企画・DX・情シスが協調する共通AI基盤** を構築します。

### ■ コンセプト：

### **「ナレッジ × ワークフロー × エージェント × セキュリティ」

による、証券業務に最適化された再現性・継続性のある業務変革**

* ナレッジの構造化 =  
  市場レポート・投信資料・過去提案書・社内規程等を  
  **横断的に検索・再利用できる構造化ナレッジ基盤**
* ワークフローの明示化 =  
  「調査 → 要約 → 提案書ドラフト → レビュー → 登録」といった  
  **連続した営業業務フローをノーコードで定義・可視化**
* エージェント化 =  
  「提案資料作成エージェント」「ナレッジ検索エージェント」「FAQボット」等の  
  **業務専用AIエージェントによる自律的・対話的な業務実行**
* セキュリティ統制 =  
  Azureベースの閉域構成・顧客テナント導入を前提とした  
  **金融機関水準のセキュリティ・ガバナンス**

# **6. AIWorkforce（AIエージェントプラットフォーム）の概要**

## 6.1. 全体構成

* **業務自動化エンジン**
  - ノーコードで定義可能な **AIワークフローエンジン**
  - 営業・法務・バックオフィスなど複数部門を横断する共通基盤
* **AIエージェント基盤**
  - ユーザーの自然言語指示を理解し、最適なワークフローを自動選択・実行するエージェント
  - 不足情報のヒアリング、再生成、修正反映までを対話的に実行
* **ナレッジポータル**
  - 社内に散在する提案書、レポート、社内文書を一元的に蓄積・検索
  - キーワード検索＋ベクター検索による高精度な類似資料検索
* **文書処理機能**
  - PDF・Word・Excel・PowerPoint等の読解・要約・生成・比較・抽出・マスキング
  - 文書を構造化データとして再利用可能にする仕組み
* **インテグレーション**
  - Officeファイル、社内ポータル、基幹システム、ストレージ等との連携
  - 将来的な API 連携・Teams/Slack からのトリガー実行も見据えた設計

## 6.2. 特徴

* **営業・法人企画・DX推進部・情シスが共通で利用できる横断型プラットフォーム**
* **ノーコード／ノープロンプト**：現場部門がプロンプト設計スキルに依存せずに利用・拡張可能
* **連続した業務フロー対応**：「調査 → 作成 → チェック → 登録」まで一貫して自動化
* **ナレッジとワークフローの連携**：過去提案書・市場レポートを参照しながら新規提案を自動生成
* **エンタープライズ・金融機関向けのセキュリティ・運用基盤**

# **8. Single-Tenant SaaS 導入アーキテクチャ**

## 8.1. システム構成図（例）

```
[Client VPC（東亜セキュリティーズ様環境）]
      │  VPN / PrivateLink
      ▼
[Dedicated Tenant (AIWorkforce)]
  |-- Web / Agent Runtime（AIエージェント・ワークフロー）
  |-- Knowledge DB（ナレッジ・メタデータ・ベクター）
  |-- Workflow Engine（ジョブキュー・ワーカー）
  |-- Audit / Logging（監査・操作ログ）
  |-- Integration Layer（社内システム連携API）
```

## 8.2. 特徴

* Microsoft Azure を基盤とした **クラウドネイティブ構成**
* 顧客専用テナント or Single-Tenant SaaS により、他社データと物理・論理的に完全分離
* Private Link / VNet による閉域網構成で、社内ネットワークとセキュアに接続
* Azure OpenAI / Azure AI Search / Azure Document Intelligence 等のマネージドサービスを活用
* Terraform 等による IaC 運用、監視・ログ収集（Azure Monitor / App Insights）に対応

# **10. 主要ユースケース（東亜セキュリティーズ様向け）**

### ■ ユースケース1：リテール営業向け提案資料自動生成

* 顧客属性（年齢・資産状況・投資方針 等）と市場環境を入力すると、  
  **最適な投信・商品を組み合わせた提案ストーリーと説明資料を自動生成**
* 投信レポート・市場ニュース・社内マテリアルを横断的に参照
* 営業担当者は生成されたドラフトを微修正するだけで提案可能

### ■ ユースケース2：法人営業の提案書ナレッジ活用

* 過去の法人向け提案書・案件情報を **データレイク化・インデックス化**
* 顧客業種・規模・目的を指定することで、**類似案件の成功パターンやスライドを高速検索**
* 過去事例を踏まえた提案書ドラフトをAIが生成し、提案準備時間を大幅削減

### ■ ユースケース3：支店スタッフ向けFAQボット

* 商品内容・取引ルール・社内規程等に関する問い合わせに対し、  
  **支店スタッフ向けAI FAQボット** が即時に回答
* 根拠となる原文（マニュアル・規程）の該当箇所も同時に提示

### ■ ユースケース4：調査・レポート作成支援

* 調査部・企画部が作成する市場レポート・社内分析資料の要約・展開
* 外部レポート・論文等からの情報抽出と、自社向けサマリー自動生成

# **13. 投資対効果（ROI試算）**

## 13.1. ROI計算式

```
ROI =（年間削減工数 × 工数単価 − 投資額）÷ 投資額
```

## 13.2. 試算例（イメージ）

* 対象：リテール・法人営業部門における提案準備・調査・資料作成業務
* 年間削減工数：3,000〜5,000時間相当（PoC結果を踏まえ算定）
* 工数単価：8,000円（例）
* 投資額：2,000万円（初期＋1年目運用費のイメージ）

→ **ROI = 1.2〜2.0（120〜200%）相当の投資対効果ポテンシャル**  
　※実際の数値は、PoC結果・対象範囲・導入スコープに応じて精緻化します。
