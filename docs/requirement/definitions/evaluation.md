# 評価指標（要件定義書）

## 評価観点

1. 網羅性
   - 必要な要件が漏れなく記載されているか。

2. わかりやすさ・明確性
   - 要件が曖昧さなく明確に定義されているか。

3. トレーサビリティ
   - ビジネス要求との対応関係が明確か。

4. 実装可能性
   - 技術的に実装可能な要件定義になっているか。

5. テンプレ準拠度
   - 章構成・見出しが大きく崩れていないか。

## スコアリング

- 各観点について 1〜5 点でスコアリングすること。
- 1: 要件定義として不十分
- 3: 一般的な要件定義書として許容できるレベル
- 5: 開発に移せる高品質なレベル

## 合格基準

- 以下を満たす場合に `pass` とする。
  - 総合評価（全観点の平均）が 4 以上
  - 「網羅性」「わかりやすさ・明確性」がいずれも 4 以上
