# 評価指標（提案書 / PoC計画書）

## 評価観点

1. 課題とのフィット感
   - クライアントの課題・背景に対して、提案内容が的確に対応しているか。

2. わかりやすさ・ストーリー性
   - 非エンジニアのステークホルダーにも理解しやすい説明になっているか。
   - 背景 → 課題 → 解決策 → 効果 の流れが自然か。

3. インパクト・説得力
   - 提案の「意味」「効果」が、経営層にも伝わる表現になっているか。
   - KPI / ROI のイメージが示されているか。

4. 構成・網羅性
   - template.md の章立てに沿っており、大きな抜け漏れがないか。
   - スケジュール・体制・リスクなど、意思決定に必要な情報がカバーされているか。

5. テンプレ準拠度
   - 章構成・見出しが大きく崩れていないか。
   - 読み手が他の案件の提案書と比較しやすい形になっているか。

## スコアリング

- 各観点について 1〜5 点でスコアリングすること。
- 1: 説得材料として弱い
- 3: 一般的な提案書として許容できるレベル
- 5: 経営層にも強く響くレベル

## 合格基準

- 以下を満たす場合に `pass` とする。
  - 総合評価（全観点の平均）が 4 以上
  - 「課題とのフィット感」「インパクト・説得力」がいずれも 4 以上
