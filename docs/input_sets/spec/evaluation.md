# 評価指標（機能仕様書）

## 評価観点

1. 正確性
   - input.md に書かれている事実・要件を正しく反映しているか。
   - 推測や憶測で重要な要件を追加していないか。

2. 網羅性
   - input.md に書かれた要件・制約を、どこかの章で必ず触れているか。
   - テンプレートの章立てに大きな抜けがないか。

3. 構成 / ロジック
   - 章の構成が論理的で、読み手が迷子にならないか。
   - 各章の役割が明確か。

4. 読みやすさ
   - 文が冗長すぎず、箇条書きを適切に使っているか。
   - 専門用語が適宜説明されているか。

5. テンプレ準拠度
   - template.md の見出し・章立てを守っているか。
   - 不要な章を勝手に削除・変更していないか。

## スコアリング

- 各観点について 1〜5 点でスコアリングすること。
- 1: 非常に不十分
- 3: 実務でギリギリ使えるレベル
- 5: 十分に高品質

## 合格基準

- 以下をすべて満たす場合に `pass` とする。
  - 総合評価（全観点の平均）が 4 以上
  - 「正確性」「テンプレ準拠度」がいずれも 4 以上
