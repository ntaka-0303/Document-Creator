# **5. プロジェクト体制**

## 5.1 体制図（例）

* クライアント側と LayerX 側の **2つのライン** を明確に分け、それぞれの縦の関係（上位〜現場）と、同じレイヤー同士の横の関係が分かるように体制図を記載します。
* 以下は、Mermaid 記法を用いた体制図の例です（実際のプロジェクトに合わせてノード名・関係線を編集してください）。  
  **左側にクライアント、右側に LayerX** を横並びに配置し、それぞれの中では縦方向に体制を表現します（クライアントと LayerX の間に上下関係はありません）。

```mermaid
graph TD
  %% クライアント側（左カラム）
  subgraph Client[クライアント側]
    C_EX[経営層・ステアリングコミッティ]
    C_PM[プロジェクトマネージャー]
    C_BIZ[業務チーム（営業企画／法人企画／バックオフィス等）]
    C_IT[情報システム部・DX推進部 等]
    
    C_EX --> C_PM --> C_BIZ --> C_IT
  end

  %% LayerX 側（右カラム）
  subgraph LX[LayerX 側]
    L_EX[エグゼクティブスポンサー／アカウント責任者]
    L_PM[プロジェクトマネージャー]
    L_CONS[業務コンサルタント／導入コンサルタント]
    L_ENG[AIエンジニア／テクニカルチーム]
    
    L_EX --> L_PM --> L_CONS --> L_ENG
  end

  %% 横の関係（同じレイヤー同士を連携させて横並びを維持）
  %% ※ Mermaidではリンクで繋ぐとランクが揃いやすくなります
  C_EX --- L_EX
  C_PM --- L_PM
  C_BIZ --- L_CONS
  C_IT --- L_ENG
```
