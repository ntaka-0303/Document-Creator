# LayerX「AIWorkforce」機能レポート（AIインプット用）

---

## 1. 概要とコンセプト

AIWorkforce は、企業内のさまざまな「文書を起点とした知的業務」を自動化・高度化する **エンタープライズ向け AI プラットフォーム** である。

- ChatGPT などの汎用チャットではなく、**実務のワークフロー** に組み込むことを前提としたプロダクト
- 経理・法務・営業・人事など、**複数部門をまたぐ横断領域（ホリゾンタル）** をターゲットにしている
- 「企業とともに成長する」ことをコンセプトに、**使えば使うほど精度が上がる仕組み** を志向
- PoC 止まりの RAG や「人がプロンプトを工夫して使うだけ」の利用ではなく、**連続した業務フロー全体を AI に委ねる** 方向性

初期段階から以下のような前提が置かれていると推測される。

- 単なる Chat UI の企業版は作らない
- Prompt スキル頼み・属人化する形は避ける
- 実務インパクトの大きいドキュメント処理に集中する

このため、AIWorkforce は **ノーコード／ノープロンプト** を掲げ、ユーザーがプロンプトを設計しなくても業務に使えるような設計になっていると考えられる。

---

## 2. 機能一覧（俯瞰）

AIWorkforce の機能を大きく以下のカテゴリに分類する。

1. 業務自動化エンジン
   - AIワークフロー
   - AIエージェント基盤
2. 文書入出力・処理機能
   - 対応フォーマット
   - 読解・要約・生成・抽出・比較・マスキング
3. モジュール群・テンプレート
   - ファイル入出力、LLMプロンプト、検索、出力処理、加工
4. ナレッジ活用機能
   - ナレッジポータル
   - ベクター検索・関連キーワード・ランキング
5. インターフェース / UX
   - Web UI、ワークフロー実行画面、レビュー・フィードバック機構
6. インテグレーション
   - Office 系ファイル
   - 社内ポータル・基幹システム
   - メール・チャット・ストレージ（将来拡張含む）
7. エンタープライズ基盤
   - セキュリティ
   - 運用・可用性
   - Azure ベースの技術スタック

## 3. 業務自動化エンジン

### 3.1 AIワークフロー

AIWorkforce の中核機能は **「AIワークフロー」** である。

- 特定業務の手順を **ノーコードで定義できるワークフローエンジン**
- 各社の業務に合わせて、モジュールやプロンプトテンプレートを組み合わせて設定
- 例：
  - 「PDF 取込 → 要約 → Excel転記 → メール通知」
  - 「契約書読み込み → リスクチェック → レポート作成」

特徴:

- 各ユースケースごとにシステム開発するのではなく、**GUIで組み替えて使い回せる**
- 現場主導で「自分たちの業務フロー」を AI に教えられる形
- ベストプラクティス化されたプロンプト・モジュールにより、ユーザー個人の Prompt スキルに依存しない

### 3.2 AIエージェント基盤

アップグレードにより、AIWorkforce は **「AIエージェント基盤」** を持つようになっている。

- 「エージェント」と「ワークフロー」を組み合わせる構造
- エージェント側：
  - ユーザーの自然言語の要望を理解する
  - 必要なワークフローを選択・実行
  - 不足情報をユーザーに質問して埋める
  - フィードバックを反映して再生成を行う
- ワークフロー側：
  - 個別タスクを安定的に実行
  - 決まった手順・品質を担保

これにより、次のような **連続した業務** を一貫して支援できる。

- 調査 → 調査レポート作成 → 再調査・修正 → 提案書フォーマットへの落とし込み
- 過去提案書検索 → 類似案件分析 → 提案書ドラフト作成 → 修正反映

エージェントは、業務ごとの専用エージェントとして構成されることが想定される。

- 例：
  - 「提案資料作成エージェント」
  - 「契約書チェックエージェント」
  - 「見積書処理エージェント」

## 4. 文書入出力・処理機能

### 4.1 対応フォーマット

AIWorkforce は以下のような主要フォーマットに対応していると考えられる。

- PDF（検索可能・スキャン／OCR含む）
- Word
- Excel
- PowerPoint
- その他テキスト/画像ベースの文書

目的は、**非構造データ（文書）を構造化データに変換し、業務で再利用できるようにすること**。

### 4.2 文書処理の主な機能

1. **ドキュメント読解・要約**
   - 長大な資料から概要やポイントを抽出
   - マニュアル・規程・調査レポートなどに対応

2. **文書生成**
   - 過去事例や入力情報を踏まえた新規文書のドラフト生成
   - 提案書・稟議書・契約書ドラフトなど
   - 社内フォーマットに近い形で出力

3. **データ抽出・転記**
   - 文書中の項目や表データを抽出し、ExcelやCSV形式で出力
   - 見積書・請求書・決算書などから必要項目を取り出し
   - 基幹システムや会計システム向けの形式に整形

4. **文書比較・レビュー**
   - 契約書ドラフトなどを社内ルール・ガイドラインと照合
   - リスクとなりうる条項を検出・指摘
   - レビュー補助用コメント・サマリーの出力

5. **機密情報のマスキング**
   - 個人情報や機密情報を検知・自動マスキング
   - 社外共有前の匿名化・秘匿処理

6. **マルチラウンド対話**
   - ユーザーが追加質問・修正指示を行うと、再処理を行う
   - 「もう少し短く」「この観点を強調」「この用語は使わない」などの指示を反映

これらにより、「読んで・理解し・要約し・別形式に書き換える」といった人間の知的作業の多くを AI に委譲できる構造になっている。

## 5. モジュール群とテンプレート

### 5.1 モジュールの種類

AIWorkforce のワークフローは、多数のモジュール（処理部品）を組み合わせて構成される。

主なモジュール例：

- **ファイル入出力系**
  - PDF／Officeファイルの読み込み
  - OCR・レイアウト解析
  - テキスト・表データ抽出

- **LLMプロンプト系**
  - 要約・分類・レポート生成
  - Q&A・文章生成
  - 事前に調整されたプロンプトテンプレートを利用

- **検索・参照系**
  - 社内のファイルストレージ・ナレッジベースを横断検索
  - キーワード検索＋ベクター検索のハイブリッド
  - 類似度スコア・関連キーワード提案など

- **出力処理系**
  - Excel / CSV / PowerPoint / Word への出力
  - メール送信
  - 社内システム API へのデータ送信

- **フィルタ・加工系**
  - 日付フォーマットの統一
  - 数値の正規化
  - 機密情報のマスキング
  - 社内ルールに応じた文言・形式の調整

- **対話エージェント系**
  - ユーザーとの対話管理
  - フィードバックの受け取りと再実行
  - 不足情報の確認質問

### 5.2 プロンプトテンプレート

- 各種タスク（要約／分類／チェック等）について、**ベストプラクティスのプロンプトがテンプレ化** されていると推測される。
- ユーザーは「やりたいこと」を選ぶだけで、裏側では最適化されたプロンプトが LLM に渡される。
- これにより、プロンプト品質に依存する性能ムラを抑え、**再現性の高い運用** が可能。

## 6. ナレッジ活用機能（ナレッジポータル）

### 6.1 ナレッジポータルの役割

AIWorkforce は、単なるワークフローエンジンではなく、社内の非構造情報を蓄積・活用する **「ナレッジポータル」** 的な役割も持つ。

- 社内に散らばる文書・レポート・過去提案書・ナレッジを集約
- 従業員が自然文で検索・参照できるインターフェースを提供
- 新しく生成されたアウトプット（要約等）もナレッジとして再利用可能

### 6.2 検索機能の特徴

- キーワード検索 + ベクター検索の組み合わせ
- AND / OR / フレーズ検索だけでなく、「あいまい検索」や意味ベースの類似検索も可能
- 検索結果には関連度スコアを付与し、並び順を最適化
- 関連キーワードのレコメンド表示により、探索的な情報発見を支援

### 6.3 ナレッジとワークフローの連携

- ナレッジポータルに蓄積されたデータは、ワークフローやエージェントから参照可能
- 例：
  - 「過去提案書を参照して新規提案を作る」
  - 「類似案件の事例を検索し、分析レポートを生成」
- 社内の知見が **「一回きりの成果物」ではなく、次の業務のコンテキストとして再利用される**。

## 7. ユーザーインターフェースと人間の関与

### 7.1 UI/UX

- ブラウザベースの Web アプリとして提供
- 典型的な操作イメージ：
  - ファイルをドラッグ＆ドロップで投入
  - ワークフローやエージェントをメニューから選択
  - 実行結果がダッシュボード上に一覧表示される
- 実行結果に対してプレビュー表示やダウンロード、再実行等が可能

### 7.2 出力の根拠表示

- AI の出力に対し、「どの部分がどの原文に基づいているか」を辿れるようにする設計が想定される
  - 要約の元になった原文箇所のハイライト
  - 抽出した項目がどこの行・セルから取られたか参照可能
- これにより、ユーザーは AI の結果を **検証・信頼** しやすくなる

### 7.3 人間のレビューとフィードバック

- ユーザーが出力結果を確認・修正できる UI が存在
- 修正内容や「正解」を AIWorkforce 側に蓄積し、次回以降の処理に活かす方向性
  - 項目値の補正
  - 判定ラベルの正誤
  - 期待されるフォーマット

- イメージとしては、
  - 「AIが初稿を作る → 人がレビュー・修正 → 修正履歴を AI が学ぶ → 次回以降の精度が上がる」というループ
- 完全にモデルを再学習するというより、**In-Context 的に知識を増やしていく** アプローチと推測される

## 8. サポートする業務領域・用途

AIWorkforce がターゲットにしている主な業務領域と用途の例は以下の通り。

### 8.1 営業・提案業務

- 過去提案書の横断検索・分析
- 顧客・案件条件を入力すると、過去事例を踏まえた提案書ドラフトを生成
- スライド検索エージェントによる「欲しいスライド」の高速抽出
- 効果：
  - 提案準備（事例調査・資料作成）の時間を大幅削減
  - 営業担当者の品質平準化

### 8.2 法務・契約業務

- 契約書ドラフトのリスクチェック
- 社内基準との不整合や抜け漏れの指摘
- 契約書要約の自動生成
- 将来的には契約書作成〜審査までの一連のフローへの適用が想定される

### 8.3 経理・財務・バックオフィス

- 見積書・請求書・発注書・決算書などの読み取りとデータ化
- 資産登録や会計システムへの入力に必要な項目抽出
- 複雑・多様なフォーマットの明細構造理解と自動分類
- 効果：
  - 1件あたりの処理時間を半分以下に削減した事例
  - 年間数千〜数万時間規模の工数削減ポテンシャル

### 8.4 人事・総務

- 履歴書や評価レポートの要点抽出
- 会議議事録の要約
- 社内規程や FAQ への自動回答

### 8.5 調査・研究・新規事業

- 大量のレポート・論文・レポート資料からの情報抽出
- 市場調査レポートや技術調査レポートの自動生成
- 新規事業の事例・競合調査などへの応用

### 8.6 行政・公共

- 申請書類のデジタル化
- 議事録作成
- 条例・規程の検索・要約

※ 公開されている情報から推測される用途も含む。

## 9. 導入事例（概要）

以下は代表的な導入・活用事例の抜粋である。

### 9.1 メガバンク A 社（営業提案領域）

- 過去提案書ナレッジの横断検索・再利用基盤として AIWorkforce を導入
- 営業担当者が過去事例を探し、提案資料を作る時間を大幅削減
- 提案準備時間の 5〜9 割を削減する効果が示されたとされる
- 追加で、エージェントによるスライド検索機能も本格運用開始

### 9.2 総合商社 B 社（ナレッジプラットフォーム）

- 社内の知的資本を共有するプラットフォームに AIWorkforce を採用
- ナレッジコンテンツ作成の効率化（既存資料から要約を自動生成）
- 社員向けナレッジ検索の UX 改善（曖昧検索・関連度順・関連キーワード）
- 数千〜数万人規模の従業員が利用する環境でも運用

### 9.3 リース会社 C 社（見積書処理）

- 多様なフォーマットのリース見積書から資産情報を抽出し、台帳登録する業務に導入
- 担当者は見積書をアップロードするだけで、AI が Excel・CSV まで自動生成
- トライアル段階で 80% 超の高精度、現場担当者の 90% 以上がメリットを実感
- 本格運用で年間 1 万時間以上の工数削減を見込む

### 9.4 SIer / コンサル企業との協業

- 大手 SIer の AI 導入支援プログラムにおける主要ツールとして採用
- 参加企業が AIWorkforce を用いて自社データで PoC を行い、そのまま本番展開のイメージを持ちやすくする
- AIWorkforce チームが伴走し、アイデア出し〜PoC 構築までを短期間で支援

## 10. 技術的基盤

### 10.1 インフラストラクチャ

AIWorkforce は、クラウドネイティブな構成で運用されている。

- 基盤クラウド：Microsoft Azure
- コンテナ実行環境：Azure Container Apps などのマネージドサービス
- IaC：Terraform によるインフラ構成管理
- 非同期処理：ジョブキュー＋ワーカー構成（Celery 等）

Azure 採用の理由（推定）：

- Azure OpenAI との統合
- エンタープライズ顧客（特に金融系）の Azure 採用率の高さ
- セキュリティ要件への適合、および顧客テナント内設置の柔軟さ

### 10.2 アプリケーション構成

- フロントエンド：Next.js
- バックエンド：Python（FastAPI + GraphQL など）
- 処理ワーカー：Python ベースのジョブワーカー

Python を中核にすることで、

- R&D の検証コード → プロダクトへの取り込みが容易
- LLM / ドキュメント処理系ライブラリの利用がしやすい

### 10.3 主なマネージド AI サービス

AIWorkforce のドキュメント処理・検索・生成には、主に次のようなサービスが組み込まれていると推測される。

- Azure OpenAI Service
  - GPT 系モデルで要約・生成・分類・Q&A を実行
- Azure AI Document Intelligence（旧 Form Recognizer）
  - PDF/画像からの OCR、レイアウト解析、テーブル・フィールド抽出
- Azure Cognitive Search（Azure AI Search）
  - インデックス構築・全文検索・ベクター検索
- Azure Cosmos DB
  - 文書メタ情報やワークフロー関連データ、ベクターなどの保存

### 10.4 ネットワークとセキュリティ構成（技術的）

- Private Endpoint / Private Link を活用した閉域網構成
- アプリケーション・DB・AI サービス間の通信を VNet 内に閉じる
- 入口には WAF 付きのフロントサービスや Firewall を設置
- 監視・ログ収集は Azure Monitor / Application Insights 等を活用

## 11. 競合比較と差別化ポイント（推定）

### 11.1 他ツールとの比較軸

1. 対象領域
   - 垂直特化（例：契約書AI、経理AI） vs 横断型プラットフォーム
2. UI/UX
   - プロンプト前提 vs ノーコード/ノープロンプト
3. 業務フロー対応
   - 単発タスク対応 vs 連続する業務フロー全体
4. システム連携
   - アウトプットを人がコピペ vs 直接基幹システム連携
5. セキュリティ・導入形態
   - 完全 SaaS のみ vs 顧客テナント導入も視野
6. 日本語最適化
   - 英語中心 vs 日本語業務前提

### 11.2 AIWorkforce の差別化ポイント

- **業務横断型のプラットフォーム**
  - 法務・経理・営業・人事など、複数部門で同じ基盤を活用可能
  - ナレッジやモジュールの再利用効率が高い

- **ノーコード／ノープロンプト**
  - ワークフロー構築は GUI ベース
  - Prompt 設計力に依存せず、現場主導で AI活用を拡張できる

- **連続した業務フロー対応**
  - 部分最適ではなく、「調査→作成→チェック→登録」まで一貫
  - エージェント＋ワークフローで複数ステップをカバー

- **社内システムへの深い統合**
  - Office ファイルとの連携に加え、基幹システム・社内ポータルと接続
  - AI のアウトプットがそのまま業務システムに反映される設計

- **エンタープライズ特化のセキュリティ**
  - Azure ベース・閉域網・顧客テナント設置など
  - 特に金融機関の導入実績が、その水準の高さを示唆

- **日本語文書への強さ**
  - 日本語の長文・複雑な会計・契約文書への適用実績
  - 日本企業向けの UI / サポート

## 12. エンタープライズ対応（セキュリティ／運用）

### 12.1 データセキュリティ

- 入出力データはクラウド内で閉じた形で処理
- LLM プロバイダ側にも「学習に利用しない」前提のサービスプランを利用
- 保存データの暗号化、バックアップ、権限管理を実装

### 12.2 アクセス管理

- 企業の認証基盤（例：Azure AD や各種 SSO）との連携を想定
- 組織・ロールに応じた権限分割
- ナレッジポータル上の閲覧制御（部署別、プロジェクト別など）

### 12.3 可用性・運用

- マネージドコンテナ基盤によるオートスケール
- ジョブキュー方式による負荷平準化
- 監視とアラートによりシステム状況を常時把握

### 12.4 カスタマイズと拡張

- 個社向けのモジュール（基幹システム連携など）を組み込む余地
- API を通じた他社サービスとの連携（将来拡張を含む）
- PoC → 段階的な本番展開という導入パターンを前提

## 14. まとめ（AI エージェント向け要約）

- AIWorkforce は、**企業の文書系業務を横断的に自動化する AI プラットフォーム**。
- 中核は、ノーコードで構成可能な **AIワークフロー** と、ユーザー要望を理解しワークフローを呼び分ける **AIエージェント基盤**。
- PDF・Office 文書などから読み取り・要約・生成・抽出・レビュー・マスキングまで一連の機能を提供。
- モジュールとテンプレートにより、非エンジニアでも高度な LLM 処理を活用可能。
- 社内ナレッジを集約する **ナレッジポータル** を持ち、検索・再利用・二次活用を支援。
- 営業提案・契約書レビュー・見積書処理・経理・人事・調査など、多様な業務で利用されている。
- 技術基盤は Azure 上に構築されており、セキュリティ・可用性・拡張性を重視。
- 競合に対する強みは、「横断型」「ノーコード／ノープロンプト」「連続フロー対応」「深いシステム連携」「エンタープライズ・日本語特化」など。
- 将来は、ユースケース拡大、外部サービス連携、エージェント高度化、グローバル展開、他プロダクトとのシナジーが進むと考えられる。

AI エージェントがこのレポートをインプットとして利用する場合は、

- 機能カタログとして
- ユースケースの例示用として
- 技術・アーキテクチャ設計の参考として
- 競合比較・PoC プラン策定の素材として

活用できるように構造化してある。
