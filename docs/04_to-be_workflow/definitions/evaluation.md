# 評価指標（To-Beワークフロー）

## 評価観点

1. 改善効果の明確性
   - As-Isとの比較で改善ポイントが明確に示されているか。

2. わかりやすさ・可視化
   - 業務フローが図やステップで理解しやすく表現されているか。

3. 実現可能性
   - 現実的に実装・運用可能なワークフローになっているか。

4. 課題解決度
   - 既存の課題が適切に解決されているか。

5. テンプレ準拠度
   - 章構成・見出しが大きく崩れていないか。

## スコアリング

- 各観点について 1〜5 点でスコアリングすること。
- 1: ワークフロー定義として不十分
- 3: 一般的なワークフロー文書として許容できるレベル
- 5: 実装に移せる高品質なレベル

## 合格基準

- 以下を満たす場合に `pass` とする。
  - 総合評価（全観点の平均）が 4 以上
  - 「改善効果の明確性」「実現可能性」がいずれも 4 以上
